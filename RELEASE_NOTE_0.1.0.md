# Release Note — v0.1.0

> 发布日期：2026-06-17

sql2java-workflow 是基于 AI Agent 的 Oracle PL/SQL → Spring Boot + MyBatis 端到端转译系统。本版本围绕跨平台兼容、质量门控、代码规范、自定义规约、测试体系等方向迭代，主要发布特性如下。

---

## ✨ 发布特性

1. **支持用户自定义代码规约**：新增 `--spec` 参数，可用 Markdown 规约文件按章节覆盖内置 Java 代码规范。

2. **新增质量门控与跨包去重阶段**：流水线由 7 阶段扩展为 9 阶段，新增翻译完成率、review 评分、测试通过率等质量门控，并支持跨包去重与公共模块抽取。

3. **跨 Schema 校验分级**：区分阻断级与告警级问题，告警项需用户确认后继续推进；fix 阶段统一回流至 review，路由更稳定。

4. **refName 重载命名规范统一**：重载子程序采用统一命名规则，并通过跨包方法索引对接包间调用，不再依赖命名猜测，避免裸名撞重载。

5. **Windows 跨平台兼容**：文件操作统一封装，不再强制依赖 bun/node；verify 阶段在无 Maven/JDK 环境时自动跳过编译与测试，不阻断整体流程。

6. **advance 校验与路径写入加固**：自动修复结构类问题以减少无谓拒绝，文件写入路径按区域分类拦截，防止 Java 源文件误写到产物目录。

7. **运行进度可视化**：通过 todowrite 维护各阶段进度，进度条在侧边栏固定展示，流程状态更清晰。

8. **新增三层测试体系**：单元测试、集成测试与基于真实 Agent 的执行点测试，覆盖核心引擎与关键流程。
